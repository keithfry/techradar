The Sequence Opinion #848: The Agent’s Hands: CLI or MCP?

The most important question in agentic software is not "Which model?" but "What can the model touch?" A language model by itself has no hands, but with tools, it becomes an operator that can interact with the world. Two candidates for bridging the model and the world are the Command-Line Interface (CLI) and the Model Context Protocol (MCP), representing different philosophies on tool interfaces. CLI advocates a Unix process-based approach, while MCP proposes a structured, discoverable, typed tool interface with protocol, schemas, resources, prompts, permissions, and client-server architecture.
