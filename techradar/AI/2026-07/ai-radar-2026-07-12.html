sqlite-utils 4.1

This 4.1 release for sqlite-utils introduces several key enhancements and bug fixes. The `insert` and `upsert` commands can now accept a `--code` option to generate rows directly from Python code blocks instead of just file paths. Users can also override the automatically chosen column type using the new `--type` flag, which is useful for preserving formats like ZIP codes. Functionality added includes dropping indices by name via `table.drop_index(name)` and allowing the `query` command to read SQL from standard input. Finally, the transform mechanism now supports switching a table's strict mode between strict and non-strict settings using `strict=True/False`.

Forget typosquatting; slopsquatting is the software supply chain threat created by AI coding tools

Slopsquatting is a novel software supply chain attack utilizing large language model (LLM) hallucinations to inject malicious code via fictitious open-source packages. Attackers exploit LLMs' tendency to suggest plausible, but non-existent, package names, circumventing traditional security measures like those used against misspellings. These malware-filled packages can be incorporated directly into a developer's codebase through AI assistance, remaining undetected for extended periods. While proprietary models show lower hallucination rates than open-source ones, all LLMs are vulnerable to this threat. Developers must verify all AI-recommended dependencies using automated checks to mitigate the expanding attack surface.
